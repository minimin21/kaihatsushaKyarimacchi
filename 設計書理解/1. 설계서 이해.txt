* 원본 건드리지말고 복사해서 작성하기

* 검색 > 등록 > 상세 > 수정 > 삭제 순으로 만드는 것이 좋다.

* 파일 이름 명 예시) A1000001-ユーザー管理-検索-基本設計書-20260429-朴政珉.

* A1000001에서 A는 API (좀 더 기능 설명이 들어간) A대신 G가 들어가면 화면 설계서라는 뜻.

* A와 G 둘다 설계서 따로 만드는 것이 좋다.

* A1000001 에서 A바로 옆 숫자 1의 의미는 메뉴마다 다르다. 마지막 숫자 1은 기능에 따라 다르다.
  예를 들어 프로젝트 모집 메뉴의 경우 4를 쓰고, 검색 설계서를 만들 때는 A4000001 또는 G4000001.
 
* 위 설명 및 아래를 참고하여 파일 이름을 지으면 된다.
   - 유저정보 (ユーザー管理) : 1 / 개발자 찾기 (求人管理) : 2 / 프로젝트 모집 (求職管理) : 3
   - 등록 : 1 / 수정 : 2 / 삭세 : 3 / 상세 : 4 / 검색 : 5

* 설계서는 일본어로 작성해야 하고, 숫자를 사용할때도 일본어 자판으로 사용해야 한다. (숫자 모양이 한글과 다름)

* 설계서에서는 です 또는 ます 사용 하지 않는다.
 ~だ  또는 단어로 문장을 끝낸다.

* 예시 설계서 내용 그대로 쓰지 않고, 설계자 본인 마음대로 내용 작성해도 됨.
  단 누구나 알아 볼 수 있도록.

* 기본 설계서 작성 후 상세 설계서를 작성하게 되는데, 기본설계서는 개발자가 아닌 사람도 보면 알 수 있도록 만들어야 함.
  상세 설계서는 개발자가 보고 어떤 기능을 넣어야 하고, 어떤 버튼을 누르면 어떤 반응이 일어나는 등을 알 수 있도록 자세하게 만들어야 함.

* 画面レイアウト탭에 화면 그리는건 画面レイアウト 참고용.PNG 를 보고 그리면 됨.

* 画面項目一覧탭 설명또한 3-2. 画面項目一覧 참고용.PNG를 보고 참고하여 작성하면 됨.
  面レイアウト에 있는 번호에 대한 설명 하나하나를 画面項目一覧 에 작성하는 것임.

*  画面レイアウト참고용.PNG처럼 해도 좋고, 아니면 동봉된 txt파일 2번 레이아웃을 이쁘게 다듬고, 일본어로 바꿔서 스크린샷으로 찍어 붙여도 됨. 대신 번호 넣어줘야 함. 画面項目一覧탭에서 설명해야하기 때문.
